## Orchestrate AI agents from a single chat interface

Octo loads AGENT.md files from your projects, connects to MCP servers, routes tasks to the right agent via a supervisor pattern, and proactively reaches out when something needs attention.

## Core Features

<CardGroup cols={3}>
  <Card
    title="Multi-Agent Supervisor"
    icon="diagram-project"
  >
    Routes requests to the right specialist — project workers, standard agents, or deep research agents.
  </Card>
  <Card
    title="Proactive AI"
    icon="bell"
  >
    Heartbeat timer and cron scheduler let Octo reach out first — no prompt needed.
  </Card>
  <Card
    title="Telegram Transport"
    icon="paper-plane"
  >
    Full bidirectional Telegram bot sharing the same conversation thread as the CLI.
  </Card>
  <Card
    title="Skills Marketplace"
    icon="puzzle-piece"
  >
    Reusable prompt modules with dependency management. Search, install, and extend.
  </Card>
  <Card
    title="MCP Integration"
    icon="plug"
  >
    Connect any MCP server. Live management — add, remove, reload without restart.
  </Card>
  <Card
    title="Context Management"
    icon="brain"
  >
    Three layers of protection against context overflow. Auto-compaction and manual controls.
  </Card>
  <Card
    title="Embeddable Engine"
    icon="cube"
  >
    Use `octo.core` as a library in your own services. No env vars, no CLI — just `OctoEngine` and `OctoConfig`.
  </Card>
</CardGroup>
